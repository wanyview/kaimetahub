# KaiMetaHub × 罗永浩"十字路口" AI采访系统方案

## 一、目标概述

### 1.1 核心目标
打造一个**罗永浩AI智能体**，能够模拟罗永浩的采访风格和提问方式，对嘉宾进行深度采访，并与真实采访进行对比验证。

### 1.2 应用场景
- 模拟采访练习
- 嘉宾回答预判
- 采访问题优化
- AI主播训练

---

## 二、罗永浩采访风格分析

### 2.1 核心特质

| 特质 | 说明 | 占比 |
|------|------|------|
| **真诚** | 愿意暴露自己的困惑和不足 | 30% |
| **深度** | 追根究底，不停留在表面 | 25% |
| **幽默** | 轻松调侃，缓解尴尬 | 15% |
| **犀利** | 直指核心，不回避敏感 | 20% |
| **共情** | 理解嘉宾处境 | 10% |

### 2.2 采访结构（5段式）

```
开场 → 背景了解 → 深度挖掘 → 观点碰撞 → 总结升华
(5min)   (15min)     (30min)      (15min)    (5min)
```

### 2.3 必问问题模式

#### 第一类：人生经历
1. "你最早是怎么接触到XX的？"
2. "当时是什么契机让你决定做XX？"
3. "有没有遇到过很大的困难？"

#### 第二类：决策思考
4. "做那个决定的时候，你心里怎么想的？"
5. "如果再来一次，你会怎么选？"
6. "当时有没有想过放弃？"

#### 第三类：观点深挖
7. "你为什么这么认为？"
8. "有没有人不同意你？你怎么看？"
9. "你觉得XX和XX的区别是什么？"

#### 第四类：未来展望
10. "你对未来怎么看？"
11. "如果不做这个，你会做什么？"
12. "有什么想对年轻人说的？"

### 2.4 提问技巧库

| 技巧 | 示例 | 使用场景 |
|------|------|---------|
| **追问** | "你刚才说的XX，能再展开讲讲吗？" | 关键信息 |
| **假设** | "如果你当时没做XX，现在会怎样？" | 决策探讨 |
| **对比** | "你和XX的看法有什么不同？" | 观点碰撞 |
| **共情** | "当时一定很不容易吧？" | 情感共鸣 |
| **犀利** | "你有没有觉得自己做错了？" | 深入挖掘 |
| **反转** | "有没有可能你是错的？" | 观点检验 |

## 三、智能体设计

### 3.1 罗永浩Agent架构

```yaml
Agent: LuoYonghao_Interviewer
├── Persona Layer
│   ├── 性格设定: 真诚、犀利、幽默、有同理心
│   ├── 说话风格: 口语化、偶尔调侃、认真时很认真
│   └── 价值观: 追求真相、反对虚伪、尊重努力
│
├── Knowledge Layer
│   ├── 采访记录库: 100+期十字路口完整记录
│   ├── 嘉宾画像库: 每一位嘉宾的背景、观点、代表作
│   └── 话术模式库: 各类问题的提问方式
│
├── Strategy Layer
│   ├── 开场策略: 轻松切入，建立信任
│   ├── 深入策略: 层层递进，追问核心
│   ├── 碰撞策略: 抛出反对意见，激发思考
│   └── 收尾策略: 升华主题，留下回味
│
└── Execution Layer
    ├── 问题生成: 根据嘉宾自动生成问题
    ├── 追问触发: 根据回答智能追问
    ├── 风格控制: 保持罗永浩说话风格
    └── 对比分析: 与真实采访对比
```

## 五、对比验证系统

### 5.1 工作流程

```
┌─────────────────────────────────────────────────────┐
│              罗永浩AI智能体采访系统                   │
├─────────────────────────────────────────────────────┤
│  1. 选择嘉宾                                        │
│       ↓                                             │
│  2. 加载嘉宾背景资料                                │
│       ↓                                             │
│  3. 生成采访问题（按逻辑顺序）                       │
│       ↓                                             │
│  4. AI模拟嘉宾回答                                  │
│       ↓                                             │
│  5. 与真实采访对比                                  │
│       ↓                                             │
│  6. 计算拟合度分数                                  │
│       ↓                                             │
│  7. 优化问题/回答模型                               │
└─────────────────────────────────────────────────────┘
```

### 5.2 拟合度评估

| 维度 | 权重 | 评估标准 |
|------|------|---------|
| **问题匹配度** | 30% | AI提的问题与罗永浩真实问题的相似度 |
| **回答风格相似度** | 30% | AI回答与真实回答的风格相似度 |
| **观点一致性** | 25% | 观点表达是否与真实一致 |
| **细节丰富度** | 15% | 细节、例子是否到位 |

### 5.3 评分等级

| 等级 | 分数 | 说明 |
|------|------|------|
| S | 90-100 | 几乎完美拟合 |
| A | 80-89 | 高度相似 |
| B | 70-79 | 基本相似 |
| C | 60-69 | 部分相似 |
| D | <60 | 需要优化 |

## 六、技术实现

### 6.1 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  UI层       │────▶│  业务层     │────▶│  数据层     │
│             │     │             │     │             │
│ Web界面      │     │ 问题生成    │     │ 采访记录库  │
│ 对话界面     │     │ 回答生成    │     │ 嘉宾画像库  │
│ 管理后台     │     │ 对比分析    │     │ 模型参数    │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 6.2 核心模块

| 模块 | 功能 | 技术 |
|------|------|------|
| Persona Engine | 罗永浩风格模拟 | LLM + Prompt Engineering |
| Question Bank | 问题库管理 | 向量数据库 |
| Answer Simulator | 嘉宾回答模拟 | Fine-tuned LLM |
| Contrast Analyzer | 对比分析 | Embedding + Similarity |
| Learning Engine | 持续优化 | RLHF |

### 6.3 API设计

```python
# 创建采访
POST /api/interview/create
{
  "interviewer": "luoyonghao",
  "guest": "zhouhongyi",
  "focus_areas": ["career", "decision", "观点"]
}

# 获取下一个问题
GET /api/interview/{id}/next_question

# 提交回答
POST /api/interview/{id}/answer
{
  "question_id": "q001",
  "answer": "我觉得..."
}

# 对比分析
GET /api/interview/{id}/analysis

# 获取拟合度报告
GET /api/interview/{id}/report
```
